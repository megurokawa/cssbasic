CSS Syntax

The CSS rule consists of two main parts: the selector, and one or more statements.

selector {declaration1; declaration2; ... declarationN }

The selector is usually the HTML element that you need to change the style.

Each statement consists of a property and a value.

Property is the Style Attribute that you want to set.Each property has a value.Attributes and values are separated by a colon.

selector {property: value}

The purpose of this line of the code below is to define the text color in the h1 element as red while setting the font size to 14 pixels.

In this example, h1 is the selector, color and font-size are the attributes, and red and 14px are the values.

h1 {color:red; font-size:14px;}

Different Wording and Unit of Value

In addition to the word RED, we can also use the hexadecimal color value # ff0000:

p { color: #ff0000; }

To save bytes, we can use the abbreviation of CSS:

p { color: #f00; }

We can also use RGB values in two methods:

p { color: rgb(255,0,0); }
p { color: rgb(100%,0%,0%); }

Notice that when using RGB percentages, percentage signs should be written even when the value is zero.

But in other cases you do not need to do that.

For example, px units do not need to be used after 0 when the size is 0 pixels, since 0 is 0, no matter what the unit is.

Remember to Put Quotation Marks

Tip: If the value is a few words, give the value plus quotes:

p {font-family: "sans serif";}

Multiple statements:

Tip: If you want to define more than one statement, you need to separate each declaration with a semicolon.The following example shows how to define the middle paragraph of a red text.

The last rule does not require a semicolon because the semicolon is a delimiter in English, not an end sign.

However, most experienced designers add a semicolon to the end of each statement, which has the benefit of minimizing the possibility of errors as you add or subtract statements from existing rules.

p {text-align:center; color:red;}

You should describe only one attribute per line, which enhances the readability of the style definition, like this:

p {
  text-align: center;
  color: black;
  font-family: arial;
}

Space and Case

Most style sheets contain more than one rule, while most rules contain more than one statement.

The use of multiple declarations and spaces makes it easier to edit style sheets:

body {
  color: #000;
  background: #fff;
  margin: 0;
  padding: 0;
  font-family: Georgia, Palatino, serif;
  }

Likewise, unlike XHTML, CSS is not case-sensitive, as it does not affect the CSS's work on browsers.However, there is an exception: The class and id names are case-sensitive if it comes to working with HTML documents.
